Batch的相关知识

在深度学习的数值优化过程中，Batch（通常指 Mini-batch）是指在单次迭代（Iteration）中用于计算梯度并更新模型参数的一组样本子集。

从数学和计算机架构的角度来看，Batch 的引入主要解决了计算效率与优化质量之间的权衡问题。

2. 计算效率：矩阵并行化 (Parallelism)

这是在计算机科学层面引入 Batch 的核心动力。现代处理器（尤其是 GPU）在处理运算时具有以下特性：

  • SIMD (Single Instruction, Multiple Data)：GPU 擅长同时对大量数据执行相同的指令。

  • 内存带宽与计算比：从显存读取权重参数到计算单元的开销很高。如果每次只处理一个样本（Batch Size = 1），权重的读取时间将远超计算时间，导致算力闲置。

矩阵表达：

通过将 个输入向量堆叠成一个矩阵 ，我们将 次单独的“矩阵-向量”乘法转化为了一次高效率的**“矩阵-矩阵”乘法**（GEMM）。这不仅利用了 GPU 的并行张量核心，还摊薄了读取权重的内存开销，极大地提升了吞吐量。

3. 对泛化性能的影响 (Generalization)

Batch Size 的选取不仅仅是计算问题，还涉及优化动力学：

  • 小 Batch (Small Batch)：梯度估计的方差较大（噪声多）。这种随机性具有类似于模拟退火的效果，可以帮助优化过程跳出损失平面上那些窄而深的“尖锐极小值”（Sharp Minima），从而更有可能落在“平坦极小值”（Flat Minima）区域。研究表明，平坦区域通常具有更好的泛化性能。

  • 大 Batch (Large Batch)：梯度估计非常精确，优化轨迹更平滑。但在相同学习率下，大 Batch 容易导致模型收敛到距离初始点较近的局部极小值，泛化能力可能较弱。此外，它对显存（VRAM）的占用呈线性增长。

4. 相关概念辨析

  • Batch Size：每个 Batch 中样本的数量（如 32, 128, 1024）。

  • Iteration / Step：利用一个 Batch 完成一次“前向传播 + 反向传播 + 参数更新”的过程。

  • Epoch：将整个数据集完整遍历一次的过程。1 Epoch = (总样本数 / Batch Size) 次 Iterations。

总结

Batch 是连接统计推断（梯度估计）与高性能计算（矩阵加速）的桥梁。它利用并行计算提升速度，同时利用抽样噪声作为一种隐式的正则化手段，帮助模型在非凸的损失空间中寻找更优的解。

2. 无偏不代表无误差：方差的影响

虽然 Mini-batch 梯度的“平均值”指向正确的方向，但单次计算结果与真实梯度之间存在偏差，这个偏差由**方差（Variance）**衡量：

（其中 是全量样本梯度的方差， 是 Batch Size）

  • 当 较小时：方差很大，梯度含有大量噪声。反映在训练轨迹上，就是更新方向非常“抖动”。

  • 当 增大时：方差减小，估计越来越精确，收敛轨迹变得平滑。

3. 为什么“无偏”对深度学习至关重要？

如果估计是有偏的（Biased），意味着在大量的迭代之后，模型参数更新的方向会产生系统性偏差，导致模型永远无法收敛到全局最优点。

由于 Mini-batch 梯度是无偏的，我们可以确信：

  1. 大数定律支持：随着迭代次数增多，随机波动的噪声会相互抵消。

  2. 收敛性保证：在满足特定条件下（如 Robbins-Monro 算法对学习率的要求），使用这种无偏估计进行的随机优化过程（SGD）在期望意义上是收敛的。

4. 计算机实现中的权衡

在计算机中，我们不直接处理无穷大的数据流，而是通过将数据集打乱（Shuffle）并分块处理。这种做法虽然在单个 Epoch 内是不放回抽样，但在宏观的优化路径上，它依然被视作对全量梯度方向的高效、廉价的无偏近似。

简而言之，无偏性保证了方向的正确性，而 Batch Size 的选择则是在调节“噪声大小”与“计算效率”之间的平衡。
