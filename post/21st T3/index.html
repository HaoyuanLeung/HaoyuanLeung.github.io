为什么要使用指针

对于一个指针，比如create file函数里的cur，要访问其成员就必须用->运算符
因为这里需要直接修改路径上的一个对象，如果设cur是一个node类型而不是node的指针类型，那么就不能实现直接修改对象。
如果使用全局数组模拟，可以避免

一些用数组模拟实现的关键操作

recycle_bin数组

用来存储被删除的节点下标。后续删除某个节点是通过重新分配下标并重新设置该下表对应的节点内容实现的。

newcode函数

优先采用recycle_bin数组里面的下标，没有了再用别的

DeleteNodeTree函数

从当前节点出发，递归遍历其所有子节点，并将其全部放入回收站中。
注意，对于回收站数组，也即存储被删除的下标的数组，和newcode函数，我们实际上并不在意被分配给某个节点的下标具体值以及大小排序等信息是多少，因此这里无需排序。

相关的指针操作

  1. 指针的声明与初始化
    声明指针变量：
    Node* root = nullptr; (声明一个指向 Node 对象的全局指针，并初始化为空)。
    Node* cur = root; (声明局部指针并用另一个指针赋值)。
    Node* parent = root;
    Node* target = parent->children[parts.back()];
    在容器中声明指针：
    map<string, Node*> children; (声明一个值类型为 Node 指针的映射表)。

  2. 动态内存分配与释放 (new 和 delete)
    使用 new 在堆（Heap）上创建对象并返回指针：
    root = new Node(true); (创建根节点对象，将其内存地址赋给 root)。
    cur->children[name] = new Node(true); (创建新的目录节点，将其地址存入 map)。
    cur->children[fname] = new Node(false); (创建新的文件节点，将其地址存入 map)。
    使用 delete 释放指针指向的堆内存：
    delete root; (释放整棵树的根节点内存)。
    delete target; (释放移除指令中被删掉的节点内存)。
    delete pair.second; (在析构函数中，遍历并释放 map 里存的所有子节点指针指向的内存)。

  3. 通过指针访问成员 (箭头操作符 ->)
    由于 cur, root, parent, target 都是指针，所有访问 Node 结构体内部变量的操作都必须使用 ->。
    访问/修改布尔值：
    !cur->is_dir
    cur->is_dir
    访问/修改数值变量：
    cur->size
    cur->size = new_size;
    cur->cur_lr += delta;
    cur->cur_ld += delta;
    cur->ld_quota
    cur->lr_quota
    调用成员对象的函数 (间接涉及)：
    cur->children.find(part) (通过指针找到 children 这个 map 对象，然后再调用 map 的 find 方法)。
    cur->children.end()
    parent->children.erase(…)

  4. 指针的移动与赋值 (遍历树形结构)
    改变指针内部存储的地址，使其指向不同的对象。
    向下移动（指向子节点）：
    cur = cur->children[part]; (从 map 中取出一个指针赋值给 cur，使 cur 指向下一层节点)。
    cur = cur->children[parts[i]];
    cur = cur->children[name];
    parent = parent->children[parts[i]];
    重置回起点：
    cur = root; (让 cur 重新指向根节点所在的地址)。

  5. 指针的比较
    指针之间的相等性比较：
    if (cur == root) (比较两个指针是否指向同一块内存地址)。

  6. C++ 迭代器 (行为类似指针，间接相关)
    在 ~Node() 析构函数中：
    for (auto& pair : children)
    虽然这里用了 C++11 的 range-based for 循环，但底层是通过 map 的迭代器（Iterator）实现的。
    迭代器在概念和行为上非常像指针（可以进行 * 解引用和 ++ 移动）。
    pair.second 取出的是 map 中存储的 Node* 指针。
