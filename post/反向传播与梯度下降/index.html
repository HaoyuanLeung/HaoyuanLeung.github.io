具体知识补充 一

在计算机科学和深度学习框架（如 PyTorch, TensorFlow）中，求导运算主要通过**自动微分（Automatic Differentiation, AD）**完成。它既不同于数值微分，也不同于符号微分。

  • 节点 (Nodes)：表示变量（张量）。

  • 边 (Edges)：表示算子（运算）。

3. 与其他求导方式的对比

方法 原理 优点 缺点
数值微分 (Numerical) 易于实现，无需源码 存在截断误差和舍入误差； 复杂度高
符号微分 (Symbolic) 类似 Mathematica 的代数规则 得到解析表达式，精确 面对复杂嵌套函数会产生“表达式膨胀” (Expression Swell)
自动微分 (AD) 算子重载/源码转换 + 链式法则 计算精确（达到浮点精度）；效率极高 需要预先定义算子的导数规则

4. 工程实现细节

  • 雅可比向量积 (VJP, Vector-Jacobian Product)：在实际张量运算中，我们并不显式构建巨大的雅可比矩阵（Jacobian Matrix），而是直接计算梯度向量与局部梯度矩阵的乘积，极大地节省了内存。

  • 动态图 vs 静态图：

    • PyTorch (动态)：在运行代码时实时构建计算图，允许使用 Python 的控制流（如 if, while）。

    • TensorFlow 1.x (静态)：先定义完整的计算图并优化，然后再输入数据执行，效率更高但灵活性较差（TF 2.x 已默认动态）。

总结来说，计算机并不进行“感性”的推导，而是将函数解析为基础算子序列，并利用链式法则在离散的计算节点上精准地传递梯度。

具体知识补充 二

在已知“节点存储变量，边存储运算”的计算图结构基础上，自动微分（Automatic Differentiation, AD）的核心执行逻辑在于：将复杂的函数分解为基本算子序列，并利用链式法则在图中传播导数信息。

总结

自动微分的运行机制可以概括为：

  1. 分解：将函数拆解为计算图。

  2. 前向：计算并缓存所有节点的数值。

  3. 反向：基于缓存值，利用预设的局部导数规则，通过 VJP 线性组合，将“误差信号”（伴随变量）从汇点反向推送到源点。

这种方式规避了符号微分的“表达式膨胀”问题，也避免了数值微分的舍入误差与高昂成本。

4. 总结：内存里到底发生了什么？

在运行反向传播时，内存中发生的是一系列的矩阵乘法和逐元素加减：

  1. 变量节点：存储当前的数值（Tensor data）和对应的梯度数值（Tensor grad）。

  2. 运算边：存储一个指向“反向函数”的指针。

  3. 求导过程：

    • 取出一个梯度实数（或张量）。

    • 取出前向传播时缓存的输入实数（或张量）。

    • 按照算子预设的数学公式进行乘法运算。

    • 将结果写回上一个节点的梯度存储区。

所以，计算机并不是在进行“符号推导”，而是在进行“导数值的算术传递”。 它利用了微积分的连锁反应，将复杂的全局求导转化为了局部数值的乘加运算。

总结

  • 没有“模型”，只有 “张量运算的拓扑图”。

  • 没有“抽象导数”，只有 “局部导数公式在当前数值下的乘积”。
