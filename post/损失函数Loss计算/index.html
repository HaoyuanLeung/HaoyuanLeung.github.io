损失函数 用于度量模型预测值 与真实标签 之间的差异。

  • 统计学背景：通常对应于最大似然估计（MLE）。例如，交叉熵损失 (Cross-Entropy) 源自 KL 散度，用于度量两个分布的距离；均方误差 (MSE) 则假设噪声服从高斯分布。

一系列损失函数

1. 回归任务损失函数 (Regression Loss)

回归任务的目标是预测连续值，其损失函数通常度量的是预测向量 与真实向量 在欧几里得空间或曼哈顿空间中的距离。

L2 Loss / 均方误差 (Mean Squared Error, MSE)

这是最常用的回归损失，其定义为：

  • 数学形状：是一个凸的二次抛物面。

  • 梯度性质：梯度 。这意味着误差越大，梯度越大，收敛越快。

  • 统计推导：当观测噪声服从高斯分布时，对似然函数取负对数即可得到 MSE。

  • 缺点：对异常值（Outliers）极度敏感，因为平方操作会放大极大误差的权重。

L1 Loss / 平均绝对误差 (Mean Absolute Error, MAE)

  • 数学形状：呈 V 字型。

  • 梯度性质：梯度是常数（除 0 点外为 ）。这意味着无论误差多大，更新步长相对固定。

  • 鲁棒性：比 MSE 更能容忍异常值（鲁棒回归）。

  • 缺点：在 0 点处不可导，且在接近极小值时由于梯度不减小，容易在最优解附近产生振荡。

Huber Loss (平滑平均绝对误差)

Huber Loss（平滑平均绝对误差）在数学上是一个分段函数，它被设计用来平衡 MSE (L2 Loss) 的收敛效率与 MAE (L1 Loss) 对异常值的稳健性（Robustness）。

2. 分类任务损失函数 (Classification Loss)

分类任务的输出通常是概率分布。损失函数度量的是预测概率分布与真实标签分布（通常是 One-hot 编码）之间的差异。

交叉熵损失 (Cross-Entropy Loss, CE)

对于多分类问题，假设有 个类别：

  • 数学本质：衡量两个分布之间的 KL 散度。在真实分布 固定时，最小化交叉熵等价于最小化 KL 散度。

  • 统计推导：当输出服从多项式分布时的负对数似然。

  • 激活函数配合：通常与 Softmax 函数配合使用。Softmax 将模型输出压缩至 且和为 1，满足概率公理。

二元交叉熵 (Binary Cross-Entropy, BCE)

二分类问题的特殊形式：

  • 数学形状：当预测概率逼近错误标签时，损失呈对数级爆炸（趋向无穷大），这会产生极强的惩罚信号，迫使模型快速修正。

Hinge Loss (合页损失)

主要用于支持向量机（SVM）和某些特定的分类网络：

  • 数学性质：它不仅要求分类正确（符号一致），还要求分类的“信心”达到一定的间隔（Margin）。如果分类正确且信心充足（），则损失直接降为 0。

3. 损失函数的数学总结表

损失函数 对应任务 统计假设 对异常值敏感度 梯度表现
MSE (L2) 回归 高斯分布 高 随距离线性增加
MAE (L1) 回归 拉普拉斯分布 低 恒定
Cross-Entropy 分类 多项式分布 中（对置信度敏感） 误差越大梯度越陡峭
Hinge 分类 最大间隔 低 分段常数或零

一个深入的细节：为什么分类不用 MSE？

既然 MSE 能度量距离，为什么分类不直接用 MSE？

  1. 梯度消失：若分类任务最后一层使用 Sigmoid，MSE 的导数会包含 项。在模型预测严重错误（处于饱和区）时， 趋近于 0，导致梯度消失，模型“学不动”。

  2. 非凸性：在分类逻辑下，MSE 与 Softmax/Sigmoid 结合后的误差曲面往往是非凸的，存在大量局部极小值。而交叉熵在概率空间中通常能提供更良性的凸优化路径。

“无穷大”

这正是交叉熵作为损失函数的优秀特性，其设计初衷是：对“确定性的错误”施加毁灭性的惩罚。

  1. 概率空间的惩罚机制：如果模型以 的信心（）认为答案是 A，但真实答案是 B（），这意味着模型的参数状态与现实发生了剧烈冲突。负对数函数的陡峭性确保了此时会产生巨大的梯度。

  2. 最大似然估计 (MLE)：BCE 源自伯努利分布的似然函数。如果我们观测到了事件 ，但模型预测其发生的概率 ，那么观测到这组数据的概率就是 0。在统计学上，概率为 0 的事件发生了，意味着模型参数是绝对错误的。

  3. 防止梯度消失：相比于 MSE（均方误差），交叉熵在 接近错误极端值时，由于 的导数特性，依然能保持巨大的梯度信号，强制模型脱离错误区域。

总结

输出 0 或 1 并不是错的。错的是：你预测输出 1，但结果是 0；或者你预测输出 0，但结果是 1。 这种无穷大的趋势在计算中确实会带来 NaN 风险，因此在工程实现（如 PyTorch 的 BCEWithLogitsLoss）中，通常会加上一个极小的 （如 ）进行平滑，或者直接在 Logit 空间计算以保证数值稳定性。
